## Checks with no findings

- **bash 3.2 compat** — clean: case-based "maps" (`imagine:41-51`, comment says why), no assoc arrays, no `${var,,}`; `[[ =~ ]]` numeric patterns are 3.2-safe.
- **jq injection** — `--arg`/`--argjson`/`--rawfile` used consistently for user text (`q:169` argv→python json, `imagine:364-382`, `enhance_prompt:63-64`, `critique:234-237`); `jsonl_entry`'s `$i` is bound via `--arg` (`_lib.sh:37`). The only string-built jq fragment is `star_entry:136` interpolating `$i` — already regex-validated numeric at 131.
- **q-spec hard contracts** — "best-effort logging never breaks the answer" (try/except `q:217-227` ✓), "q never starts/stops the server" (`q:160` prints hint only ✓), warm-pin keep_alive guard present in both `q:165` and `imagine:60` with matching comments.
- **Reuse vs reinvention** — `_lib.sh` adoption is genuine across q/imagine/lm; the one duplicate is auto-base.sh's inline residency check (row above). `lm_status:44` lists rather than tests residency — not a duplicate.
- **`rm` usage** — `imagine:240` `rm -f "$tmp"` and `lm:67` are in-script mktemp/probe cleanup, not agent Bash calls; the trash convention targets the latter. Not flagged.
- **Comment quality** — good throughout: human-first WHY comments, no plan refs, no `[claude@]` tags, no shipped TODOs. `_lib.sh:22`'s "works forever" claim is contradicted by the numbering-drift finding, but the *comment* isn't the bug.
- **`$RANDOM` seeding** (`imagine:330`) — fine for image seeds (reproducibility, not security); always recorded.
