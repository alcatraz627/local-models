## Clean checks (verified, no issue found)

- **Happy paths**: `q --json`, `q --stream-json` emit only protocol JSON on stdout; stream frames flush per-chunk (`chunk`…`done`). `mkdir -p`, `_lib.sh` color init (gated on `[ -t 1 ]`), and the thinking trace all go to stderr or are suppressed in machine modes (bin/q:275, 286, 296-299). No stdout leak observed.
- **server_down** (preflight + mid-request OSError) → `{code:server_down}` exit 10 in both json and stream modes (bin/q:213-216, 304-306). Verified.
- **model_missing** (HTTP 404) → `{code:model_missing}` exit 11 (bin/q:300-302). Verified live with a bogus `-m`.
- **exec orphan**: `exec python3 … <<'PY'` is the final bash statement (bin/q:228); nothing executable follows. History write + done-frame happen **inside** python, so the exec doesn't orphan the tail logic. Verified.
- **stdin double-consume**: python reads its program from the heredoc and its inputs from **argv** — no `sys.stdin`/`input()` in the python block (grep clean). `--ctx -` consuming stdin does not conflict with the heredoc. `-c` reads history from the LOG file (sed/tail), not stdin — no conflict with `--ctx -`.
- **setitimer ordering**: timer is disarmed at bin/q:307 **before** the best-effort log write (319-327), so a timeout can't fire during logging. Log write is wrapped in try/except (best-effort).
- **warm detection jq**: `any(.[]; . == $m or startswith($m + ":"))` correctly matches both `gemma4-e4b-warm` and `gemma4-e4b-warm:latest`, and returns cold on `[]` (bin/lm:51-52). Verified.
- **`warm ensure` removal**: clean — zero dangling `ensure` references in bin/, docs/, config.sh. warm help text correctly documents read-only-for-machines (bin/warm:46-48).
- **lm_status_json under set -e**: `if ps_json=$(curl…)` preserves curl's exit for the branch; jq failures are caught by `|| resident='[]'` / `|| available='[]'`; `local` declarations on :42-43 assign literals (no exit-masking of a command substitution). No spurious abort observed.
- **`lm status --json` dispatch**: `status) shift; …` correctly re-checks `$1` for `--json` after the shift (bin/lm:119). Verified both branches.
- **config.sh LM_VERSION**: added, surfaced as `toolkit_version` with `${LM_VERSION:-1.0}` fallback (bin/lm:60). api_version (1) is separate. Verified `1.1` appears in output.
